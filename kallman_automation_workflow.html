Kallman Automation Workflow

Printable guide for creating, editing, testing, and pushing automation projects to production.
Simple model: Work in testing. Test it. Merge to main. Pull main on the server. Task Scheduler runs production.

Workflow 1: Create a New Automation Project

1. Start in the repo

cd C:\kwi-automations
git switch testing
git pull origin testing

2. Create the project folder

New-Item -ItemType Directory -Path "projects\PROJECT-NAME" -Force
New-Item -ItemType Directory -Path "projects\PROJECT-NAME\scripts" -Force
New-Item -ItemType Directory -Path "projects\PROJECT-NAME\config" -Force

3. Start Codex in the repo

codex.cmd

4. Prompt Codex

Use C:\kwi-automations as the project root.
Create or edit only inside C:\kwi-automations.
Create the project at projects\PROJECT-NAME.
Do not include passwords, API keys, exports, CSVs, or Excel files.

5. Test locally

.\projects\PROJECT-NAME\run.ps1
# or for Python:
py .\projects\PROJECT-NAME\scripts\main.py

6. Commit to testing

git status
git add .
git commit -m "Add PROJECT-NAME automation"
git push origin testing

7. Promote to production branch

git switch main
git pull origin main
git merge testing
git push origin main

8. Update the server computer

cd C:\Automations\Production\kallman-automation
git switch main
git pull origin main

Workflow 2: Edit an Existing Automation Project

1. Start in testing

cd C:\kwi-automations
git switch testing
git pull origin testing

2. Start Codex

codex.cmd

3. Prompt Codex

Use C:\kwi-automations as the project root.
Edit this existing project: projects\PROJECT-NAME.
Make a small, controlled change.
Do not edit outside this repo.
Do not add passwords, API keys, CSVs, Excel files, or exports.

4. Test locally

.\projects\PROJECT-NAME\run.ps1
# or for Python:
py .\projects\PROJECT-NAME\scripts\main.py

5. Review changes

git status
git diff

6. Commit to testing

git add .
git commit -m "Update PROJECT-NAME automation"
git push origin testing

7. Promote to main

git switch main
git pull origin main
git merge testing
git push origin main

8. Pull production on server

cd C:\Automations\Production\kallman-automation
git switch main
git pull origin main

Production Rules

DoDo Not
Work on testing first.Do not let Codex edit the server production folder.
Test before merging to main.Do not commit passwords, API keys, exports, CSVs, or Excel files.
Keep the server computer on main.Do not switch the server computer to testing.
Use SharePoint Data Warehouse for input/output files.Do not run long-term production from Desktop folders.
